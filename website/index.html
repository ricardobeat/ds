new-engine Overview Examples

new-engine

A small interpreter for a JS-shaped language, written in C3 0.8.0. The language is informally called DS, and its source files use the .ds extension.

What it is

new-engine is a tree-walking interpreter built over a kernel of about twelve AST nodes. Every surface feature, including let, sequencing, binary operators, method calls, type annotations, and sum types with match, desugars into that kernel before evaluation. The kernel stays small, and the language grows by adding desugarings rather than evaluator cases.

Syntax

The syntax is JS-shaped. There are let bindings, records, arrays, functions, conditional expressions, and exceptions.

let greet = fn(name) {
  "Hello, " + name
}

let user = { name: "Alice", age: 30 }
let nums = [1, 2, 3]

let msg = if user.age > 18 then "adult" else "minor"

try {
  throw "boom"
} catch e {
  "caught: " + e
}

Pattern matching

Tagged sum types are matched with match. A _ arm acts as a wildcard.

let handle = fn(msg) {
  match msg {
    Key(k) -> "pressed " + k
    Tick   -> "tick"
    _      -> "ignored"
  }
}

Reactive signals

A $-prefixed binding is a mutable reactive cell. let $x = 0 creates a signal, $x reads it, and $x = e sets it. derived builds a value that tracks its dependencies, and effect runs when they change.

let $count = 0
let doubled = derived(fn() { $count * 2 })

effect(fn() {
  print("count is " + $count)
})

$count = $count + 1

Elements and TUIs

JSX-like elements compile to a referentially-stable element tree. The tree is used to build terminal user interfaces, with <VStack> and <HStack> for layout.

let view = fn() {
  <Box border="rounded">
    <VStack>
      <Text value="hi" />
      <Text value="there" />
    </VStack>
  </Box>
}

milktea

new-engine ships with milktea, an Elm-Architecture TUI framework. A model is a record of { state, update, view }, and you hand it to run. The xray layer performs differential terminal rendering, so only changed cells are redrawn.

let app = {
  state: { count: 0 },

  update: fn(state, msg) {
    match msg {
      Inc -> { count: state.count + 1 }
      _   -> state
    }
  },

  view: fn(state) {
    <Box border="rounded">
      <Text value="count: " + state.count />
    </Box>
  }
}

run(app)

Quick start

Build the interpreter with C3, then run or format a file.

c3c build
./build/engine run hello.ds
./build/engine fmt hello.ds

For milktea TUI apps there is a separate ds host target.

c3c build ds
./build/ds app.ds

See the examples for longer programs.
